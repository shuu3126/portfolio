ブランド訴求 / タレント紹介 / オーディション導線

ABOUT

目的 / 背景 / 役割

公式サイトの刷新。ブランドの世界観を保ちながら、ニュース発信・タレント知名度向上・オーディションへの遷移を同時達成することが目的。
既存サイトはファーストビューの情報密度が高く、導線が散漫になりがちだったため、価値提案の明確化と主導線の集約を行った。

  • 私の役割： 企画 / 情報設計 / デザイン / コーディング（HTML/CSS/JS）/ 軽量モーション設計
  • 関係者： 代表1名 / タレントマネージャ1名（レビュー）
  • 納期： 約3週間（要件定義1、制作2）
KPI

KPI / 成果指標

  • 主要CTA（Audition）クリック率：+18%目標 → 初月対比 +13%（仮想比較）
  • ニュース一覧のセッション到達率：+15%目標
  • 直帰率：FVの簡素化と動画遅延読込で -12% 目安
  • CVR監視のため、クリック計測イベントを設置（GA/GTM想定）
FLOW

主要ユーザーフロー

  1. 求職者： Top（動画で世界観把握）→ Audition CTA → 募集要項 → 応募フォーム
  2. ファン： Top → Talents → 各タレント個別 → SNS誘導
  3. 関係者： Top → News → プレスリリース確認
IA/UI

情報設計 / UIルール

  • ページ構成： Hero（動画＋コピー）→ News → Talents → CTA（Audition）→ Footer
  • 階層設計： H1=ブランド命題 / H2=主要セクション / H3=カード内見出し
  • タイポ： 太めの英字見出し＋可読性の高い本文、行間1.7、文字間.02〜.04em
  • カラー： ブランド色をアクセントに、背景は白/薄灰でコントラスト確保
  • モーション： フェード・スライドは300–400msで軽量、パララックスは不使用
SPEC

使用技術 / 実装方針

  • HTML / CSS / JavaScript（バニラ）
  • 画像最適化（webp/サイズ上限）、遅延読込（loading="lazy"）、Hero動画は遅延再生
  • 共通CSSユーティリティ（余白・角丸・影・ボタン）で保守性を担保
  • アクセシビリティ：見出し階層、代替テキスト、フォーカス可視化、コントラスト比
QUALITY

SEO / パフォーマンス

  • タイトル/メタ/OGP整備、内部リンクを主導線に集約
  • キーワードは「事務所名＋公式」「タレント名」など固有名詞中心に設計
  • パフォーマンス・バジェット：LCP < 2.8s、CLS < 0.1 を目安に最適化
OPS

運用体制 / リスク管理

  • Gitでバージョン管理、Notionでタスク・WBS、Discordで日次報告
  • 更新頻度の高いNewsはテンプレ化して運用コストを最小化
  • 障害系リスク：動画CDN停止時は静止画に自動フォールバック
RETROSPECT

振り返り / 改善案

  • オーディションCV計測の粒度を強化（フォーム分岐ごとにイベント付与）
  • 多言語化（英語）に備え、見出し長の伸縮に耐えるUIへアップデート
